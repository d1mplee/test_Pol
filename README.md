# polymarket-research

Исследовательский пайплайн, который **эмпирически проверяет, есть ли на Polymarket
edge**, который переживёт транзакционные издержки — прежде чем вкладывать хоть цент.

Это **не бот**. Здесь нет размещения ордеров, кошелька, ключей и денег. Только
чтение публичных API и оффлайн-анализ. Цель — принять решение go/no-go по числам,
а не по надежде.

## Почему так, а не сразу бот

Данные по рынку жёсткие: ~84% трейдеров в убытке, боты дают 75% объёма,
арбитражное окно закрывается за ~2.7с. Прибыль определяется скоростью и издержками,
а не «угадал событие». Поэтому сначала измеряем, а не торгуем.

## Установка

```bash
pip install -r requirements.txt
# при SSL-перехвате в сети добавь:
#   --trusted-host pypi.org --trusted-host files.pythonhosted.org
```

## Использование

```bash
# 1. Собрать данные (метаданные + история цен), read-only, без аккаунта
python -m pm.collect --limit 500 --closed both --fidelity 720

# 2. Прогнать все три гипотезы + итоговый отчёт
python -m analysis.report

# или по отдельности:
python -m analysis.h1_yesno_arb      # арбитраж Yes+No (живой снапшот стакана)
python -m analysis.h2_longshot_bias  # favorite-longshot bias (resolved-рынки)
python -m analysis.h3_mean_reversion # mean-reversion после издержек
```

Результаты (CSV + графики) складываются в `output/`.

## Структура

| Файл | Назначение |
|---|---|
| `config.py` | URL публичных API, параметры сети, пути |
| `pm/client.py` | read-only HTTP-клиент (Gamma/CLOB/Data), backoff, парсинг строковых полей |
| `pm/collect.py` | сбор рынков и истории цен на диск (идемпотентно) |
| `pm/costs.py` | **модель издержек** — taker-fee по формуле Polymarket + спред |
| `analysis/h1..h3` | три гипотезы о неэффективностях |
| `analysis/report.py` | сводка go/no-go |

## Ключевые ограничения данных

- Минутная история доступна только для **активных** рынков. Для **resolved**
  рынков CLOB отдаёт только 12ч-свечи (`fidelity>=720`) — при меньшем вернёт `[]`.
- Тонкая история по уже закрытым рынкам бесплатно недоступна (нужен платный
  сервис типа PolyHistorical или чтение логов Polygon) — вне рамок этого этапа.

## Что дальше

Только если хотя бы одна гипотеза даёт **устойчивый положительный net-edge после
издержек** → следующий этап: paper-trading движок (симуляция исполнения по
реальному стакану, всё ещё без денег). Реальный капитал — не раньше этого.
